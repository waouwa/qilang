Appendix E

 

The Type Security of the L Type System

 

We define a primitive data structure (p.d.) as a symbol, boolean, number, string, character, or list or tuple of p.d.s. The expression ‘|- e : A’ is taken to mean ‘e : A is provable in the L type system’.  

 

Our proof of the correctness of the L type system is covered in 3 theorems.  In the first theorem, we demonstrate that if e is p.d. and |- e : A, then e : A.  In the second theorem, we demonstrate the subject reduction property, namely;

 

if |- e : A and e Þ e* then |- e* : A.

 

The third theorem is a consequence of both these theorems; namely that if |- e : A, then the evaluation of e to a p.d. e* will produce an expression of type A.  

 

Notice that it is not true that if e Þ e* and |- e* : A then |- e : A. The expression (if (= 1 0) a “a”) evaluates to “a”, but although |- “a” : string, it is not a theorem that (if (= 1 0) a “a”) : string.

 

Theorem 1: if e is a p.d. and |- e : A then e : A.

 

Proof:  Suppose e is a p.d. and |- e : A; then e is either

 

(a)     (a)    a primitive expression (symbol, boolean, number, string or character), or

(b)     (b)    a list of p.d.s, or

(c)     (c)     a tuple of p.d.s.

 

Suppose (a), then the type of e is determined by t, so that e : string iff string Î t(e) etc.  The only type rule in L for proving the type of e is the Primitive Rule which establishes e : A only if  A Î t(e).  Hence if |- e : A then A Î t(e) and so e : A.

 

Suppose (b), then e is a list and the proof proceeds by induction.  Let the ultimate length (u) be defined as follows.  

 

u(x) = 1 where x is a primitive object.

For any list l, u(l) = S(u(i)) where i is an element in l.

 

Base Case: u(e) = 0

 

then e = []; according to the Primitive Rule, which is the only applicable rule, |- e : "A (list A).  By the inhabitation rule for lists (chapter 10);  

 

"A(e : (list A) « ("x  element (x,e) É x : A))[1][1]

 

Since ~$x element(x,e) then e : "A: (list A).

 

Inductive Case: the theorem holds for all lists l where u(l) £ n.

 

Let e = [X | Y] be a list expression of ultimate length n + 1.  Then if e has a type under Qi, it has the type (list A) for some A and so |- e : (list A).  This can only be established by the Cons (right) Rule and so |- X : A and |- Y : (list A).  By the inductive hypothesis we conclude that X : A and Y : (list A).  In which case [X | Y] : (list A) and so e : (list A).

 

Suppose (c), then e is a tuple (@p x y).  Let the tuple size (s) be defined as follows.  

 

s(x) = 0 if x is not a tuple.

s(@p x y) = 1 + s(x) + s(y).

 

The proof is by induction on the size of s(e).

 

Base Case: s(e) = 1

 

Suppose |- e : A; then for some B, C, A = (B * C) and |- (@p x y) : (B * C).  But if |- (@p x y) : (B * C) then |- x : A and |- y : B. But since s(x) = s(y) = 0, x and y are not tuples and are primitive, x : B and y : C.  By the inhabitation rule for tuples:

 

((@p x y) : (B * C)) « ((x : B) & (y : C))

 

Since the RHS of this equivalence is true, then (@p x y) : (B * C) and so e : A.

 

Inductive Case: the theorem holds for all tuples t where s(t) £ n.  

Suppose s(e) = n + 1 and |- e : A; then for some B, C, A = (B * C) and |- (@p x y) : (B * C).  But if |- (@p x y) : (B * C) then |- x : A and |- y : B.  Also s(x) £ n and s(y) £ n and so by the inductive hypothesis, x : B and y : C. By the inhabitation rule for tuples:

 

((@p x y) : (B * C)) « ((x : B) & (y : C))

 

Since the RHS of this equivalence is true, then (@p x y) : (B * C) and so e : A.

 

Theorem 2: if |- e : A and e Þ e* then |- e* : A.

 

The proof proceeds by cases. The reduction of e to e* must be one of the rules #0-16 in the semantics for L.

 

By rule #1; then e = (= x y) and if |- e : A then by the Equality Rule A = boolean.  By rule #1, e* = true and by the Primitive Rule |- true : boolean.

 

By rule #2; then e = (= x y) and if |- e : A then by the Equality Rule A = boolean.  By rule #2, e* = false and by the Primitive Rule |- false : boolean.

 

By rule #3; then e = (= x y) and suppose |- e : A.  By rule #3, e* = ^ or e* = Ä and by the rules for error conditions, e * : A.

 

By rule #4; then e = ((l p x) y).  Assume |- e : A, then by the Applications Rule for some B

|- (l p x) : B ® A

|- y : B

 

Either match(p, y) = Ä or match(p, y) ¹ Ä.   Assume match(p, y) = Ä, then e* = Ä and |- e* : A (since |- Ä : (" A A)).

 

Assume match(p, y) ¹ Ä, then either

 

(a) p is a constant and p = y and so e* = x.  We have |- (l p x) : B ® A which by the Patterns Rule is provable only if |- p : B and p : B |- x : A.  But if |- p : B then p : B |- x : A iff |- x : A.  Hence |- e* : A.

 

 

(b) p is a variable; then e* = [x]y/p.  We have |- (l p x) : B ® A which by the Patterns Rule is provable only if

 

z : C |- z : B where z is any arbitrary name and C is a fresh type variable.

z : B |- [x]z/p : A  

 

Note: the sequent z : C |- z : B is trivially soluble by unification of B with the fresh C.  In this case the Patterns Rule just acts like the Abstractions Rule in Simply Typed Lambda calculus.  

 

But if z : B |- [x]z/p : A for any arbitrary z, then certainly y : B |- [x]y/p : A and given |- y : B then |- [x]y/p : A and so |- e* : A.

 

(c)  p is a pattern of the form (cons v w) or (@p v w); then since match(p, y) ¹ Ä then match(p, y) is a set s of bindings.  Each element <v, b> of s is an association of a variable v with a value b.  We know from the Patterns Rule that (l p x) : B ® A is provable only if

 

1.        1.       N1 : A1,…Nn  : An |- p* : B;

2.        2.       p* : B |- x* : A;  

 

where N1,…Nn are fresh names and A1,…,An are fresh type variables, p* results from p by replacing all the variables x1,..,xn in p by N1,…,Nn and x* results from x by replacing all the free variables from x1,..,xn by N1,…,Nn.

 

p* : B |- x* : A  states that assuming an arbitrary substitution instance of the variables in p (i.e. p*) to be of type B, that same substitution applied to the free variables in x produces an object (i.e. x*) that can be proved to be of type A.   Since match(p, y) succeeds y is a substitution instance of p.  So we have

 

s(p) : B |- s(x) : A

 

But s(p) = y and s(x) = e*.  So we have

 

y : B |- e* : A

 

But given |- y : B then |- e* : A

 

By rule #5; then e = (if x y z); and e* = y or e* = z or e* = ^.  Assume that |- e : A, then this is provable by the Conditional Rule and so |- y : A and |- z : A. Since |- ^ : A, then |- e* : A.

 

By rule #6; then e = (let x y z), suppose |- e : A.  Then by the Local Rule for some B, where x* is fresh;

 

|- y : B and x* : B |- zx*/x : A

 

Here e* = ((l x z) y); by the Applications Rule e* : A if for some B;

 

|- y : B and

|- (l x z) : (B ® A)

 

By the Patterns Rule, |- (l x z) : (B ® A) just when, where x* is fresh and C is a fresh type variable.

 

x* : C |- x* : B and

x* : B |- zx*/x : A

 

Since x is a variable the first sequent is easily soluble (unify C with B). So |- e* : A if both the following are provable.

 

|- y : B and

x* : B |- zx*/x : A

 

By hypothesis these are both provable, hence |- e* : A.

 

By rule #7; then e =  (cons x y) and |- e : A and either e* = ^ or e* = (cons ßx ßy).  If e* = ^ then |- e* : A.  Suppose e* = (cons ßx ßy). We write ‘e Þn e*’ when e can be normalised to e* using n rewrite rules.  The proof of subject reduction is by induction on n. Suppose n = 0; then e = e* and the proof is immediate.  Suppose that subject reduction holds when n rules are used; i.e. we assume if |- e : A and e Þn en, then |- en : A as an inductive hypothesis.  We wish to show |- e : A and e Þn+1 en+1`, then |- en+1 : A. In this case we need to show only that each individual rule of our semantics preserves subject reduction where no further normalisation is needed above what is performed in the rule itself. Such a proof is essentially nothing more than a reiteration of the cases already cited.  In the case of rule #7, if no normalisation of x or y is required, then again e = e* and the proof is immediate.

 

What our reasoning establishes is that the presence of the ß symbol is irrelevant to the subject reduction property.  Provided every rule preserves subject reduction in every case where no further normalisation is needed above what is performed in the rule itself, then subject reduction will obtain for the system overall.  In future we will invoke this argument to banish the ß by the phrase “by induction on the order of rewriting”.

 

By rule #8; then e =  (@p x y) and |- e : A and either e* = ^ or e* = (@p ßx ßy).  By induction on the order of rewriting, we drop the ß and consider only e* = ^ or e* = (@p x y).  If e* = ^ then |- e* : A.  If e* = (@p x y) then the proof is immediate.

 

By rule #9; then e = (++ x) and |- e : number.  Either e* = ^ or e* = (1 + ßx). If e* = ^ then certainly |- e : number.   If  ßx is a number then 1 + ßx is a number and by the Primitive Rule, |- 1 + ßx : number and so |- e* : number.

 

By rule #10; then e = (- - x) and |- e : number.  Either e* = ^ or e* = (ßx - 1). If e* = ^ then certainly |- e : number.   If  ßx is a number then ßx - 1 is a number and by the Primitive Rule, |- ßx - 1 : number and so |- e* : number.

 

By rule #11; then e = (cases x1 … xn) and |- e : A. Either ßx1 = Ä or ßx1 ¹ Ä.  By induction on the order of rewriting, we drop the ß and consider only the cases x1 = Ä or not x1 = Ä.

 

If x1 = Ä then e* = (cases … xn).  By the Cases Rule, |- e : A just when each xi in (cases x1 … xn) is such that |- xi : A.  So certainly it must be true that |- (cases … xn) : A and therefore |- e* : A.

 

If x1 ¹ Ä, then e* = x1 and since by the Cases Rule for each xi in (cases x1 … xn), |- xi : A then |- x1 : A and so |- e* : A.

 

By rule #12; then e* = Ä and since |- Ä : (" A A)), subject reduction holds.

 

By rule #13; then e = (where x y) and e* = (if x y Ä).  Suppose |- e : A, then by the Guard Rule, |- x : boolean and |- y : A. By the Conditional Rule to prove |- e* : A, it suffices to prove |- x : boolean and |- y : A and |- Ä : A. By hypothesis, the first two are provable and the third follows from |- Ä : (" A A)).

 

By rule #14; in that case e is a combinator expression, (Y (l v x)) and e* = [x]v/(Y  (l v x)).  Assume e : A, then (Y (l v x))  : A and this is provable only if v : A |- x : A.  Consider the proof tree for v : A |- x : A.  For every subgoal of the form v : A, the assumption v : A will be used to solve it.   Now replace this subgoal by (Y  (l v x)) and drop the assumption v : A.  By the Combinator Rule, this is provable if v : A |- x : A, which by hypothesis is provable.  Hence if there is a proof of v : A |- x : A then there is a proof of [x]v/(Y  (l v x)) : A and so |- e* : A.

 

By rule #15; in that case e is an application (x y). Suppose |- e : A and e* =  ^ then |- e* : A.  Suppose e* =  (ßx ßy).  By induction on the order of rewriting, we drop the ß and consider only the case (x y) and the proof is trivial.

 

By rule #16; then e = e* and the proof is immediate.

 

Theorem 3: if |- e : A then, if ße is a p.d., then ße : A.

 

Assume |- e : A and ße is a p.d.. We write ‘e Þm e*’ when e can be normalised to e* using m rewrite rules. The proof is by induction on m. If e Þ0 ße, then e is a normal form and by theorem 1, ße : A..  Assume the theorem holds for n rewrites.  Assume e Þn+1 ße; then for some e*, e Þ e* Þ n ße.  By theorem 2 we know that if |- e : A then |- e* : A, and by the inductive hypothesis, we have if  |- e* : A then ße : A. So |- e : A implies ße : A.
