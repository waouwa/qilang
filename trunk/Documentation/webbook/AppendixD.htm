Appendix D

 

The Semantics of L

 

This appendix describes the semantics of the pure functional language L that is Turing complete. This appendix also gives a series of recursive equations that describes how Qi can be compiled into L.  We assume as L system functions, ++ (the successor function), -- (the predecessor function), @p, if, let, cases, where and cons.  L contains two special symbols; ^ designates the error condition and Ä the special error condition generated from unsuccessfully matching an extended abstraction to an input. A primitive object of L is a string, number, character, boolean or symbol which is not used as a system function.

 

The Syntax of L

 

A primitive object is a formula of L.

( ) is a formula of L.

^ and Ä are formulae of L.

If x, y and z are formulae of L, so is (if x y z), (cons x y), (@p x y), (where x y), (= x y), (x y).

If x is a variable and y and z are formulae of L, then (let x y z) is a formula of L.

If xi,…,xn (0 £ i £ n) are formulae of L, so is (cases xi…xn).

 

A primitive object is a pattern.

A variable is a pattern.

( ) is a pattern.                                                                                

If p1 and p2 are patterns then so is (cons p1 p2) and (@p p1 p2).

 

If p is a pattern and x is a formula of L  then the abstraction (l p x) is a formula of L.

 

If v is a variable and the abstraction (l v  x) is a formula of L  then (Y  (l v  x)) is a formula of L.

 

We assume that all patterns are linear; i.e. there are no repeated variables. Next the rules giving the operational semantics of L.

 

 

The Operational Semantics of  L

 

The symbol Þ  means “rewrites to” and ß indicates the normal form of an expression under rewriting; error(x) indicates x evaluates to Ä or ^.  Rules are applied in order of appearance.

 

Rule #1. (= x y) Þ true if  ßx = ßy  and not error(ßx)

  

Rule #2. (= x y) Þ false if  ßx ¹ ßy

and not error(ßx) and not error(ßy)

 

Rule #3. (= x y) Þ ^ if  error(ßx) or error(ßy)

 

Rule #4. ((l p x) y) Þ let s be match(p, ßy); if s = Ä then Ä else sub(s, x)

 

Rule #5.  (if x y z) Þ  if ßx = true then y else if ßx = false then z else ^

 

Rule #6.  (let x y z) Þ ((l x z) y)

 

Rule #7.  (cons x y) Þ  if error(ßx) or error(ßy) then ^

else (cons ßx ßy)

 

Rule #8.   (@p x y) Þ  if error(ßx) or error(ßy) then ^

else (@p ßx ßy)

 

Rule #9.   (++ x) Þ  if ßx is a number then 1 + ßx else ^

 

Rule #10.  (- - x) Þ if ßx is a number then ßx - 1 else ^

 

Rule #11. (cases x1 … xn) Þ if ßx1 = Ä then (cases … xn) else ßx1.

 

Rule #12. (cases) Þ ^

 

Rule #13. (where x y) Þ (if x y Ä)

 

Rule #14. (Y  (l v x)) Þ [x]v/(Y  (l v x))

 

Rule #15. (x y) Þ  if error(ßx) or error(ßy) then ^ else (ßx ßy)

 

Rule #16. x Þ x

 

The equations for match and sub are as follows.

Match #1. match(x, x) = {}

Match #2. where x is a variable; match(x, y) = {<x, y>}         

Match #3. match((cons x y), (cons w z))

= let s1 be match(x, w), let s2 be match(y, z);

                                if s1 = Ä or s2 = Ä then Ä  else s1 È s2

Match #4. match((@p x y), (@p w z))

= let s1 be match(x, w), let s2 be match(y, z);

                                if s1 = Ä or s2 = Ä then Ä  else s1 È s2

Match #5. match(x, y) = Ä in all cases not covered by Match #1-#4

 

Sub#1 sub({}, x) = x

Sub#2 sub({<x, y>} È S, z) =  sub(S, [z]y/x)
